# Quick Task 260507-g17: Standardize body text size to 1.125rem across design system

**One-liner:** Collapsed three different "body" font-sizes (20px / 16px / 18px-arbitrary) into one canonical 1.125rem (18px) baseline; lede paragraphs and microcopy left intentionally untouched.

## Outcome

Body default, case-study MDX prose, and project/featured card descriptions now all render at `1.125rem` (18px). The two intentional ledes — home hero description and case-study summary — remain visually larger by design. All microcopy (eyebrows, footer, header, ArrowLink, timeline `<dd>`, project card titles, mobile nav) and all heading rules are unchanged.

## Edits Applied (4 surgical edits across 4 files)

### Edit 1 — `src/styles/global.css` (body rule, lines 82–83)

**Before:**
```css
body {
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
  letter-spacing: -0.01em;
  font-size: 20px;
  line-height: 28px;
}
```

**After:**
```css
body {
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
  letter-spacing: -0.01em;
  font-size: 1.125rem;
  line-height: 1.6;
}
```

Why: `20px` was bleeding into any content not styled by a more specific rule. The three preceding declarations were left untouched.

### Edit 2 — `src/pages/projects/[id].astro` (`.case-prose p` rule inside `<style is:global>`, line 132)

**Before:**
```css
.case-prose p {
  font-size: 1rem;
  line-height: 1.625;
  color: var(--color-text-secondary);
  margin-top: var(--spacing-md);
  margin-bottom: var(--spacing-md);
}
```

**After:**
```css
.case-prose p {
  font-size: 1.125rem;
  line-height: 1.625;
  color: var(--color-text-secondary);
  margin-top: var(--spacing-md);
  margin-bottom: var(--spacing-md);
}
```

Why: case-study MDX paragraphs were rendering at 16px — too small. Only `font-size` changed; `line-height: 1.625` (the case-study reading rhythm) and the color/margin declarations are preserved per the do-not-touch block.

### Edit 3 — `src/components/ProjectCard.astro` (line 57)

**Before:**
```astro
<p class="text-[18px] leading-[28px] text-[var(--color-text-secondary)]">
```

**After:**
```astro
<p class="text-lg text-[var(--color-text-secondary)]">
```

Why: Tailwind v4's `text-lg` is `font-size: 1.125rem` + `line-height: 1.75rem` (28px), which matches the previous arbitrary values exactly. Visual rendering unchanged; class names tidier and aligned to the canonical utility.

### Edit 4 — `src/components/FeaturedCard.astro` (line 56)

**Before:**
```astro
<p class="text-[18px] leading-[28px] text-[var(--color-text-secondary)]">
```

**After:**
```astro
<p class="text-lg text-[var(--color-text-secondary)]">
```

Same substitution and rationale as Edit 3.
