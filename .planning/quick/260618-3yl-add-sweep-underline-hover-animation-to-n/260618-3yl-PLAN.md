# Quick Task 260618-3yl: Add sweep-underline hover animation to nav tabs

## Goal

Give the desktop nav tabs (Work, About, Resume, LinkedIn) the same bottom-underline
hover animation used by the `ArrowLink` ("View project →") component — a left-to-right
sweep on hover — while keeping the active tab's persistent underline.

## Task

1. **Edit `src/components/Header.astro`** — replace the nav underline `<span>`'s opacity-toggle
   `::after` with the ArrowLink sweep mechanic.
   - files: `src/components/Header.astro`
   - action: Swap `after:transition-opacity` + `after:opacity-{0,100}` for a `transform`-based
     sweep: rest `after:scale-x-0 after:origin-bottom-right`, hover
     `group-hover:after:scale-x-100 group-hover:after:origin-bottom-left`,
     `after:transition-transform after:ease-in-out after:duration-300`. Active tab uses a
     persistent `after:scale-x-100 after:origin-bottom-left`.
   - verify: `npm run typecheck` passes; inactive tabs sweep an accent underline on hover;
     active tab stays underlined.
   - done: Nav hover matches the ArrowLink underline animation.

## Scope notes

- Active vs. inactive states are kept in separate `class:list` branches so no conflicting
  `after:scale-x-*` utilities land on the same element.
- The active underline mechanism changes from opacity to a static `scale-x-100` — visually
  identical at rest, but shares the inactive tabs' `::after` definition.
- Matched the ArrowLink exactly, which has no `prefers-reduced-motion` guard (consistency
  over stricter motion-reduction). Revisit if a reduced-motion fallback is wanted for both.
