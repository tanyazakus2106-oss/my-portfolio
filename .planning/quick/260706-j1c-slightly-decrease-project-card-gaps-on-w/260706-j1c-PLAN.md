# Quick Task 260706-j1c — Slightly decrease project card gaps

## Goal

Reduce the vertical gap between project cards on the Work (home) page by a modest
amount, without touching the section's outer padding.

## Context

- Card stack (`index.astro:97`) used `gap-[var(--spacing-section)]` =
  `clamp(64px, 40px + 6.39vw, 132px)` — the SAME token as the section's pt/pb
  (`index.astro:89`). So inter-card gap == section padding (up to 132px on wide
  screens), which read as too airy.

## Task

**Files:** `src/pages/index.astro` (line 97)

**Action:** decouple the card gap from `--spacing-section` and lower its ceiling:
`gap-[var(--spacing-section)]` -> `gap-[clamp(64px,40px+6.39vw,112px)]`.
- Mobile floor unchanged (64px).
- Desktop ceiling 132px -> 112px (~15% slight decrease).
- Fluid behavior preserved.

**Untouched:** `--spacing-section` token (global.css), the section pt/pb
(line 89, still 132px), the Projects heading margin.

## Verify

- `npm run build`: output CSS is valid `gap:clamp(64px,40px + 6.39vw,112px)`
  (Tailwind auto-spaced the `+` for calc validity — confirmed in dist CSS).
- `npm run typecheck`: 0 errors.
- Section pt/pb still `--spacing-section`.
- HOLD before commit — owner approval required.
