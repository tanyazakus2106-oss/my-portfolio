# Quick Task 260615-0nc: Project cards reveal one-by-one on scroll

## Problem

Project cards on the home page already use the shared `.animate-on-scroll`
pattern, but `scroll-animation.ts` assigned the stagger `--stagger-index` by
**global DOM order**. With the full-height hero pushing every card below the
fold, card #N revealed with `N × 70ms` of `transition-delay` — so lower cards
sat invisible for a growing lag before fading. Reads as sluggish, not a smooth
"one by one as I scroll" reveal.

## Approach (best practice)

Stagger is for clusters that enter together; sequential scroll reveals should
fire per-element with no accrued delay. So:

1. At init, detect which targets are within the **initial viewport** (read all
   `getBoundingClientRect` in one pass to avoid layout thrash).
2. Stagger only that initial cluster (cascade on load); give every below-fold
   target `--stagger-index: 0` → it reveals immediately when scrolled into view.
3. Tune the observer: `threshold: 0` + `rootMargin: 0px 0px -12% 0px` so each
   card triggers as it comfortably enters, not at a 5%-visible sliver.

CSS timing (500ms ease-out, 24px rise) left untouched — already matches the
project's "subtle, fast" motion guidelines.
