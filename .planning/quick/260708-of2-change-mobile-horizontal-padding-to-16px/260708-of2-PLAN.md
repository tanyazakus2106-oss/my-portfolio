<objective>
Make the site-wide `.container` horizontal gutter responsive: 16px on mobile, 24px from the `md` (768px) breakpoint up.

Currently `.container` (src/styles/global.css) applies a fixed 24px gutter (`var(--spacing-lg)`) at every viewport width. On small screens 24px is too generous; drop it to 16px (`var(--spacing-md)`) below 768px while preserving 24px at 768px and above.

Purpose: tighten mobile edge margins so content uses more of a narrow screen, without changing desktop/tablet layout.
Output: A mobile-first `.container` rule in global.css plus a matching `@media (min-width: 768px)` override, both using existing spacing tokens. One stale comment in MobileNav.astro corrected.
</objective>

<breakpoint_decision>
Use **mobile-first `@media (min-width: 768px)`** — Tailwind v4's default `md` breakpoint.

Why 768px (min-width), not `max-width: 767px`:
- The project is a mobile-first Tailwind v4 codebase. Every component's responsive behavior already keys off Tailwind's `md:` (768px) / `lg:` (1024px) prefixes — confirmed in about.astro, index.astro, projects/[id].astro, and in image `sizes` (`(min-width: 768px)`).
- global.css's only existing width media query is itself min-width based (`@media (min-width: 1200px)` for the root font size). Min-width is the established house style.
- Writing `.container` mobile-first (base = 16px, `min-width: 768px` bumps to 24px) makes the gutter transition land at exactly the same breakpoint as every `md:` utility, so the whole page reflows consistently at 768px.
</breakpoint_decision>

<gutter_coupling_audit>
Audited every place that could assume a fixed 24px gutter (task constraint). Findings:

- **FullBleedImage.astro breakout** (`width: 100vw; margin-left: calc(-50vw + 50%)`): SAFE, no change needed. The breakout only requires *symmetric* left/right padding, not the specific 24px value. `content-center = viewport-center` holds at any symmetric gutter, so the image still spans exactly 100vw at 16px.
- **Image `sizes` hints** using `calc(100vw - 3rem)` — about.astro:31, CaseImage.astro:16, projects/[id].astro:118. `3rem` = the old 48px total gutter. Below 768px the actual content width becomes `100vw - 2rem`, so these hints now slightly overestimate (by 16px). This is a harmless srcset selection hint (worst case: browser picks a marginally larger candidate — no layout or visual change). Deliberately NOT changed: an accurate fix would require inserting an extra `(min-width: 768px)` media condition into each `sizes` string to cover the 768–1024px band where 3rem is still correct — added complexity for zero visual benefit. Left as-is by design.
- **MobileNav.astro:47** comment "Top bar mirrors the header's .container (24px padding)": MobileNav uses `class="container"`, so it inherits the new responsive gutter and still mirrors the header. Only the parenthetical is now stale on mobile — corrected in Task 2.

```css
  margin-right: auto;
  /* fixed 24px gutter, matching the case-study inset */

```
