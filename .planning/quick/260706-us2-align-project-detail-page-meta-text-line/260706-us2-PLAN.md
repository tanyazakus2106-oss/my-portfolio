Static analysis completed during planning — read this before touching code. It
changes the shape of the fix and rules out the obvious-but-wrong approach.

**Both elements already share the identical `.container` geometry.**

- The single `.container` definition lives in `src/styles/global.css:234`:
  `max-width: var(--layout-max)` (= 1200px), `margin: 0 auto`,
  `padding-left/right: var(--spacing-lg)` (= 24px). There is no second
  `.container` definition and no page-level override (grep confirmed).
- Detail page (`src/pages/projects/[id].astro:62`): the meta `<p>` (line 65) is a
  direct child of `<header class="container ...">`. Its left edge = container
  inner-left = 24px gutter, 1200px cap, centered.
- Work/home page (`src/pages/index.astro:95`): the project list sits in
  `<div class="container">`; each `ProjectCard` `<article>` / `<a>` row also
  starts at container inner-left = 24px gutter, 1200px cap, centered.
- Tailwind preflight (via `@import "tailwindcss"`) zeroes UA margins, so there is
  NO hidden `<p>`/`<dd>` indent. (Note: the TIMELINE `<dd>` at line 90 already
  carries `m-0` for exactly this reason — belt-and-suspenders.)

**Conclusion:** at the CSS level the meta line's left edge and the card's
structural left edge ALREADY coincide at the 24px `.container` gutter. So the
naive fix ("add/remove horizontal padding to match") is WRONG and would introduce
a NEW misalignment.

**Most likely real cause of what the user sees (verify this):** the work-page
cards use an ALTERNATING 60/40 layout (`ProjectCard.astro`: `isReversed = index % 2 === 1`).
On NON-reversed cards (1st, 3rd, ...) the image is the left 60% column and the
uppercase role LABEL lives in the RIGHT 40% text column — so that label sits far
right of the meta line. On reversed cards the label aligns at the left. The user
is probably comparing the detail meta line to a card's LABEL text (which is on the
right for the first card), not to the card's structural left edge.

Your job: confirm the real offset empirically, then align to the user's true
intent — do NOT blindly add a gutter.
