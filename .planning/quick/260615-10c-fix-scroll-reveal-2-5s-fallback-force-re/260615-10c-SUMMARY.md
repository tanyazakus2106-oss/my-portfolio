# Quick Task 260615-10c: Fix scroll reveal defeated by 2.5s fallback — Summary

## Root cause

`scroll-animation.ts` had a fallback `setTimeout(..., 2500)` that force-revealed
**every** still-hidden `.animate-on-scroll` target — added for a zoom edge case
where IntersectionObserver never fires. On the home page the hero is full-height,
so all project cards sit below the fold. A visitor who reads the hero for >2.5s
(normal) triggered the fallback, which silently flipped all cards to `is-visible`
before they ever scrolled. Result: cards were already shown by the time the user
scrolled down — no one-by-one animation. The earlier timing fix (260615-0nc)
never touched this fallback, which is why "I don't see changes" persisted.

## Fix

`src/scripts/scroll-animation.ts` — the fallback now reveals only targets that
are **actually in the viewport** at the 2.5s mark (checks `getBoundingClientRect`
against `innerHeight`). Below-fold cards stay hidden and animate when scrolled to,
while the zoom edge case (in-view cards that never got an observer callback) is
still covered.

## Verification

- `npm run build` → Complete; confirmed `innerHeight`-based fallback is present
  in the bundled `dist/_astro/*.js`.
- `prettier --write` applied.
