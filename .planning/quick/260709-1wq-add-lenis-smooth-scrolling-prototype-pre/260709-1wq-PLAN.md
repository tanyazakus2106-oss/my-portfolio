2. Create `src/scripts/smooth-scroll.ts` as a minimal, self-contained vanilla-TS module. Match the existing script conventions (leading comment explaining intent + reduced-motion behavior, load-time guard, `document.readyState` init timing). Use exactly this structure:

```ts
// Lenis smooth-scroll — EVALUATION PROTOTYPE, wired site-wide via BaseLayout.
// Tuned to approximate the scroll feel of yrmn.me: Lenis default exponential
// ease-out with duration ~1.0s. Guarded by prefers-reduced-motion — when the
// user requests reduced motion, Lenis is NOT initialized and native scrolling
// stays fully intact. Remove this file + the BaseLayout <script> hookup to revert.

import Lenis from "lenis";
import "lenis/dist/lenis.css";

function initSmoothScroll(): void {
  if (typeof window === "undefined") return;

  // Do not smooth-scroll for users who asked to reduce motion.
  if (window.matchMedia("(prefers-reduced-motion: reduce)").matches) return;

  // autoRaf runs Lenis's internal requestAnimationFrame loop (no manual raf loop).
  // duration + default easing gives the eased, weighty feel of the reference site.
  new Lenis({
    duration: 1.0,
    autoRaf: true,
  });
}

if (document.readyState === "loading") {
  document.addEventListener("DOMContentLoaded", initSmoothScroll);
} else {
  initSmoothScroll();
}
```

Notes / guardrails:
- Do NOT add a `tailwind.config.js`, a new theme token, or any framework runtime — this is one dependency plus one TS file.
- Do NOT hand-write the recommended Lenis CSS; importing `lenis/dist/lenis.css` covers it and is Vite-processed inside the bundled Astro `<script>`.
- Leaving `new Lenis({...})` as a bare statement is intentional (autoRaf keeps the instance alive); do not assign it to an unused variable.
