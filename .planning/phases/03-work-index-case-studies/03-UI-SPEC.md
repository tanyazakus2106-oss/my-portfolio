# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Work Index & Case Studies.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Scope Note

Phase 3 delivers the portfolio's three core content surfaces: the homepage (hero + featured projects), the /work index (numbered alternating-layout project cards with scroll animations), and /projects/[slug] case study pages (structured long-form template). All components use tokens only — no hardcoded values. Placeholder MDX content is used throughout Phase 3; real case studies are added by Tanya post-phase.

Inherited from Phase 1. No new tokens introduced.

| Token | Value | Phase 3 Usage |

- Project number on work cards (`text-6xl`) is a typographic size, not a spacing token.

- Next/prev navigation arrows on case study pages: `min-h-[44px]` touch target maintained per inherited RESP-03 exception.

## Typography

Four sizes, two weights. No exceptions beyond what is declared here.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 48px | 600 (Semibold) | 1.1 | Homepage hero headline only |
| Heading | 24px | 600 (Semibold) | 1.25 | Card title, case study page H1 (project title), case study section headings (Problem / My Role / Process / Outcome), featured project card title |
| Body | 16px | 400 (Regular) | 1.6 | Case study prose, card summary text, metadata values |
| Label | 13px | 400 (Regular) | 1.0 | Skills tags, role tags, metadata keys (TIMELINE, TEAM, MY ROLE), category labels on case study header — rendered in `uppercase letter-spacing: 0.08em` (small-caps style via CSS) |

**Notes:**
- Display (48px / 600) is used for the hero headline only. It must not appear on any other element.
- The case study page H1 (project title) uses the Heading role at 24px / 600. The 48px size above plus surrounding whitespace provides sufficient page hierarchy — no additional size exception is needed.
- Section headings on the case study page (Problem / My Role / Process / Outcome) use the Heading role at 24px / 600 but rendered in muted Label style: `uppercase`, `letter-spacing: 0.08em`, `color: var(--color-text-secondary)` — matching the metadata key style. This satisfies D-14.
- Hero sub-line (role description below headline): Body role, 16px / 400.
- Availability badge / inline text: Label role, 13px / 400 uppercase.

Each project card on /work uses the project's `accentColor` frontmatter value. Scope is strictly limited to work index cards — not case study pages (per D-15).

**Accent reserved for (complete inherited list plus Phase 3 additions in bold):**
1. Active nav link indicator (Phase 2)
2. Nav link hover underline (Phase 2)
3. Footer "Get in touch" link (Phase 2)
4. Keyboard focus rings via `:focus-visible` (Phase 2)
5. **Card CTA link text on /work page** ("View case study →") — global accent, not per-project color
6. **Per-project `accentColor` on work index cards** — project number text, card left-border accent stripe or card title color (executor chooses one consistent application per card; must be documented in component comment)

**Important:** The per-project `accentColor` is scoped to work index card components only. It must be applied via an inline `style` prop (`style={{ color: accentColor }}`) — not via a Tailwind utility class — because the value is dynamic per project.

### Scroll Entrance Animation

| Property | Value | Source |
|----------|-------|--------|
| Library | Vanilla `IntersectionObserver` + CSS transitions — zero runtime dependency | D-19 |
| Animation style | Fade-up: `opacity: 0` + `translateY: 24px` → `opacity: 1` + `translateY: 0` | D-18 |
| Trigger class | Add `.is-visible` class when element enters viewport | D-19 |
| CSS initial state | `.animate-on-scroll { opacity: 0; transform: translateY(24px); transition: opacity 500ms ease-out, transform 500ms ease-out; }` | D-18, D-22 |
| CSS triggered state | `.animate-on-scroll.is-visible { opacity: 1; transform: translateY(0); }` | D-18 |
| Animation duration | 500ms ease-out | D-22 |
| Stagger delay | 50ms × item index within the same viewport batch | D-21 |
| IntersectionObserver threshold | `0.1` — element 10% visible triggers the animation | Default — subtle, doesn't require full card in view |
| IntersectionObserver rootMargin | `"0px 0px -40px 0px"` — triggers slightly before element reaches viewport bottom edge | Claude's discretion — prevents pop-in at very edge |
| What animates | (a) Work index card rows, (b) case study section blocks (Problem / My Role / Process / Outcome sections) | D-20 |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` — `.animate-on-scroll` has `transition: none; opacity: 1; transform: none;` — elements render in final state immediately | D-23, POL-01 |
| Implementation | Shared inline `<script>` in `/work` page and case study layout — or a dedicated `src/scripts/scroll-animation.ts` imported via Astro's `<script>` tag | D-19 (Claude's discretion on file location) |

## Implementation Notes for Executor

1. **Google Fonts:** The existing `wght@400;600` URL in `BaseLayout.astro` is sufficient — no Bold variant is needed. Do not change the Google Fonts URL.

2. **Token-only styling (D-17 from Phase 2):** All new components use `var(--color-*)` and `var(--spacing-*)` tokens. No hardcoded hex values or pixel values except where a specific non-token value is prescribed in this spec (e.g. `opacity: 0.4` on project number, `letter-spacing: 0.08em` for labels).

3. **Per-project accent application:** In the work index card component, read `accentColor` from the project entry and apply via `style` attribute: `style={{ color: entry.data.accentColor }}`. This is the only correct approach — Tailwind utility classes cannot be dynamically constructed from runtime values in Astro.

4. **MDX full-bleed component:** The `<FullBleedImage>` component (or equivalent name — executor's choice) must break out of the 760px column. Typical approach: `margin-left: calc(-50vw + 50%)` negative margin technique, or `width: 100vw; margin-left: calc(-1 * var(--page-padding))`. Document the chosen technique in the component and leave a usage comment in both placeholder MDX files.

5. **IntersectionObserver script:** Prefer a single shared script at `src/scripts/scroll-animation.ts` (or `.js`) imported via Astro's `<script>` tag with `is:inline` or as a module. Observe all elements with class `.animate-on-scroll`. The stagger delay is applied via `--stagger-index` CSS custom property on each element, computed at observation time.

6. **Placeholder MDX content:** Both `project-alpha.mdx` and `project-beta.mdx` must demonstrate all template sections (Problem / My Role / Process / Outcome) and include one constrained image and one full-bleed image using the custom component. Include a comment at the top of each MDX file: `{/* Template sections shown below — replace with real content */}`.

7. **No new CSS files:** All styles via Tailwind utility classes and global CSS custom properties. Component-scoped `<style>` blocks in `.astro` files are acceptable only for the `.animate-on-scroll` state transitions (which require `@media (prefers-reduced-motion: reduce)` — easier in a style block than a utility).
