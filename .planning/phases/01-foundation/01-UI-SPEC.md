# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note

Phase 1 deliverables are structural: Astro scaffold, `content.config.ts` schema, root layout shell (header + footer slots), and a Cloudflare Pages preview deploy. There are no content pages, interactive flows, or user-facing states in this phase. This contract defines the design token system that all subsequent phases inherit. Every token declared here is the single source of truth — later phases extend but do not contradict these values.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight list item gaps |
| sm | 8px | Compact element spacing, tag/pill padding |
| md | 16px | Default element spacing, paragraph margins |
| lg | 24px | Card internal padding, section sub-spacing |
| xl | 32px | Layout gaps between components |
| 2xl | 48px | Major section breaks, vertical rhythm between page sections |
| 3xl | 64px | Page-level vertical padding (top/bottom of full-width sections) |
| 4xl | 96px | Hero section breathing room, above-fold whitespace. Justification: 64px (3xl) is already reserved for page-level vertical padding on full-width sections; the hero above-fold area requires meaningfully more air than a standard section break to create the visual hierarchy that separates the identity statement from the page body. 96px = 1.5× 64px — a proportional step rather than an arbitrary value. Collapsing hero breathing room to 64px would make it visually indistinguishable from ordinary section spacing. |

Exceptions: Touch targets (nav links, CTA buttons) must be minimum 44px height regardless of visual size — achieved via `min-h-[44px]` padding, not by changing token values. This exception is required for Phase 5 mobile verification (RESP-03) and should be baked into the layout shell's header from Phase 1.

## Typography

Phase 1 implements 2 weights only: 400 (Regular) and 600 (Semibold). Weights 500 and 700 will be declared when their phases introduce them (500 in Phase 2 for nav labels; 700 in Phase 3 for the hero display heading).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (Regular) | 1.6 | All paragraph text, descriptions, case study prose |
| Heading | 24px | 600 (Semibold) | 1.25 | Section headings (h2), card titles, case study section titles |

**Notes:**
- Maximum 4 type scales total across the entire project. Label (13px / 500) and Display (48px / 700) will be added in their respective phases — do not introduce them before then.
- Body line-height is 1.6 (not the 1.5 default) to aid readability for long case study prose.
- Heading line-height is 1.25 to keep multi-line headings visually tight.

## Color

### Light Mode (primary)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Page background, all surface backgrounds |
| Secondary (30%) | `#F5F5F4` | Card backgrounds, header background (scrolled), footer background |
| Text — Primary | `#1C1C1C` | Body text, headings, nav links |
| Text — Secondary | `#6B7280` | Labels, captions, metadata, secondary descriptions |
| Border | `#E5E7EB` | Dividers, card borders, input outlines |
| Accent (10%) | `#2563EB` | Reserved for: active nav indicator, primary CTA button, link underlines, focus rings |
| Destructive | `#DC2626` | Not used in Phase 1; reserved for future phases if error states emerge |

### Dark Mode (declare in Phase 1, implement in Phase 2)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F0F0F` | Page background |
| Secondary (30%) | `#1A1A1A` | Card backgrounds, header background (scrolled), footer |
| Text — Primary | `#F5F5F4` | Body text, headings, nav links |
| Text — Secondary | `#9CA3AF` | Labels, captions, metadata |
| Border | `#2D2D2D` | Dividers, card borders |
| Accent (10%) | `#3B82F6` | Same reserved elements as light mode (slightly lighter for dark contrast) |
| Destructive | `#EF4444` | Reserved |

**Accent reserved for (complete list — do not apply accent elsewhere):**
1. Active nav link indicator (underline or dot)
2. Primary CTA button fill
3. Text link underlines (on hover)
4. Keyboard focus rings (`:focus-visible` outline)
5. Case study accent color overrides (Phase 3) — each project's `accent` frontmatter field will override this value per-page

**Implementation note:** Tailwind v4 CSS variables are declared once in `@theme {}` inside `global.css`. Dark mode variants use CSS `@media (prefers-color-scheme: dark)` as the base, overridden by a `.dark` class on `<html>` when the user toggles manually (Phase 2). Both layers must be declared in Phase 1's `global.css` even though the toggle is built in Phase 2.

## Content Schema Contract

`content.config.ts` must validate all of the following frontmatter fields at build time (Phase 1 deliverable FOUND-02). This schema is referenced here because it determines what data is available to components in all subsequent phases.

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `title` | `string` | Yes | Display title of the project |
| `slug` | `string` | Yes | URL-safe identifier; used for `/projects/[slug]` routing |
| `role` | `string` | Yes | Tanya's role on the project, e.g. "Lead UX Designer" |
| `accentColor` | `string` | Yes | CSS color value (hex); overrides accent for this project's card and detail page |
| `thumbnail` | `image()` | Yes | Astro Content Layer image reference; used in work index cards |
| `skills` | `string[]` | Yes | Array of skill tags shown on project cards |
| `summary` | `string` | Yes | 1–2 sentence description shown on project cards and in meta description |
| `publishDate` | `date` | Yes | Used for ordering projects in the work index |
| `featured` | `boolean` | No | Default `false`; `true` surfaces project in homepage featured section (Phase 3) |

## Layout Shell Contract

The root layout (`src/layouts/BaseLayout.astro`) built in Phase 1 must satisfy:

| Constraint | Value |
|-----------|-------|
| Max content width | `1200px` (centered, `mx-auto`) |
| Page horizontal padding | `24px` (lg token) on mobile; `48px` (2xl token) on tablet+; `64px` (3xl token) on desktop |
| Header height | `64px` fixed (aligns to 4xl/2 = natural rhythm) |
| Header position | `sticky top-0 z-50` |
| Header background | Dominant color at 95% opacity + `backdrop-blur-sm` for frosted effect on scroll |
| Footer background | Secondary color |
| Footer padding | `48px` vertical (2xl token) |
| `<slot />` padding top | `0` — individual pages own their own top spacing |
| Landmark regions | `<header>`, `<main>`, `<footer>` — required for accessibility (FEATURES.md: "accessibility basics must be built in from day one") |
| `lang` attribute | `<html lang="en">` |
| Viewport meta | `<meta name="viewport" content="width=device-width, initial-scale=1">` |
